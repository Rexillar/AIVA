# Security Policy (Governance)

If you discover a security vulnerability in AIVA, please follow the steps below to report it privately to the maintainers.

Reporting
- Create a private issue with the 'security' label on GitHub **or** email `security@example.com` (replace with the repository owner's secure email).
- Include a clear description of the issue and a minimal reproducible example.
- Do not post security issues publicly.

Response Process
- We'll acknowledge receipt within 48 hours.
- Triage and an initial response will take place in 72 hours.
- Fixes will be staged, reviewed, and released as soon as possible with CVE assigned when required.

Please avoid sharing details publicly until a fix is available.

Disclosure
- Coordinated disclosure is preferred. If you must responsibly disclose, please provide a reasonable grace period for the maintainers to patch the issue first.
